# Решение проблемы "Failed to fetch"

## Проблема
Ошибка "Failed to fetch" при загрузке задач означает, что браузер не может подключиться к серверу.

## Решения

### 1. Убедитесь, что сервер запущен

Проверьте, запущен ли сервер:
```bash
netstat -ano | findstr :8000
```

Если сервер не запущен, запустите его:
```bash
python main.py
```

Или используйте файл:
```bash
run_server.bat
```

### 2. Проверьте адрес в браузере

Убедитесь, что вы открыли правильный адрес:
```
http://localhost:8000
```

НЕ используйте:
- `https://localhost:8000` (без SSL)
- `127.0.0.1:8000` (может не работать из-за CORS)

### 3. Проверьте консоль браузера

Откройте консоль разработчика (F12) и проверьте ошибки:
- Если видите CORS ошибки - проверьте настройки CORS в main.py
- Если видите "net::ERR_CONNECTION_REFUSED" - сервер не запущен

### 4. Проверьте файрвол/антивирус

Файрвол или антивирус может блокировать подключение к localhost:8000.
Временно отключите их для проверки.

### 5. Перезапустите сервер

Если сервер запущен, но не отвечает:
1. Остановите сервер (Ctrl+C)
2. Запустите заново: `python main.py`
3. Обновите страницу в браузере (F5)

### 6. Проверьте файл tasks.json

Убедитесь, что файл `data/tasks.json` существует и содержит задачи:
```bash
python -c "import json; data = json.load(open('data/tasks.json', encoding='utf-8')); print(f'Задач: {len(data[\"tasks\"])}')"
```

## Быстрая проверка

Запустите в терминале:
```bash
python test_api.py
```

Если видите "API работает!" - сервер в порядке, проблема в браузере.
Если видите ошибку - проблема в сервере.

# Инструкция по формату данных для клинических задач

## Формат файла

Все клинические задачи должны храниться в файле `data/tasks.json` в формате JSON.

## Структура файла

```json
{
  "tasks": [
    {
      "id": 1,
      "title": "Название задачи",
      "patient_info": "Описание пациента",
      "complaints": "Жалобы пациента",
      "anamnesis_morbi": "Anamnesis morbi",
      "epidemiological_history": "Эпидемиологический анамнез",
      "objective_data": "Объективные данные",
      "laboratory_data": "Лабораторные данные",
      "tasks": [
        "Задание 1",
        "Задание 2",
        "Задание 3"
      ],
      "reference_answer": {
        "criteria_1_syndromes": "Эталон ответа по критерию 1",
        "criteria_2_epidemiological": "Эталон ответа по критерию 2",
        "criteria_3_objective_data": "Эталон ответа по критерию 3",
        "criteria_4_diagnosis": "Эталон ответа по критерию 4",
        "criteria_5_examination_plan": "Эталон ответа по критерию 5",
        "criteria_6_treatment_plan": "Эталон ответа по критерию 6"
      }
    }
  ]
}
```

## Описание полей

### Основная информация о задаче

- **id** (число, обязательно): Уникальный идентификатор задачи (1, 2, 3, ...)
- **title** (строка, обязательно): Название задачи, например "Клиническая задача 1"
- **patient_info** (строка, обязательно): Описание пациента, например "И., 30 лет, 25 июля обратился к врачу."
- **complaints** (строка, обязательно): Жалобы пациента
- **anamnesis_morbi** (строка, обязательно): Anamnesis morbi - история развития заболевания
- **epidemiological_history** (строка, обязательно): Эпидемиологический анамнез
- **objective_data** (строка, обязательно): Объективные данные осмотра
- **laboratory_data** (строка, обязательно): Лабораторные данные
- **tasks** (массив строк, обязательно): Список заданий для студента, обычно:
  - "Обоснование диагноза."
  - "Составление плана диагностики."
  - "Составление плана лечения."

### Эталонный ответ (reference_answer)

Объект с 6 критериями оценки:

- **criteria_1_syndromes** (строка): Ведущие синдромы заболевания
- **criteria_2_epidemiological** (строка): Эпидемиологические данные
- **criteria_3_objective_data** (строка): Характерные для данного заболевания объективные данные
- **criteria_4_diagnosis** (строка): Обоснование диагноза
- **criteria_5_examination_plan** (строка): План диагностики (обследования)
- **criteria_6_treatment_plan** (строка): План лечения

## Как добавить задачи из PDF

1. **Извлеките текст из PDF** - можно использовать онлайн-конвертеры PDF в текст или скопировать вручную
2. **Разбейте каждую задачу** на соответствующие поля
3. **Заполните эталонные ответы** по каждому из 6 критериев
4. **Добавьте задачу в массив `tasks`** в файле `tasks.json`
5. **Убедитесь, что ID уникальны** и последовательны

## Важные замечания

- Используйте `\n` для переноса строк внутри текстовых полей
- Сохраняйте файл в кодировке UTF-8
- Проверьте корректность JSON-синтаксиса перед сохранением
- Все поля обязательны, но если какого-то поля нет, можно указать пустую строку `""`
- ID должны быть уникальными для каждой задачи

## Альтернативные форматы для предоставления задач

Если у вас задачи в PDF, вы можете:

1. **Скопировать текст из PDF** и предоставить в виде:
   - Текстового файла (.txt) - каждая задача отдельным блоком
   - Word документа (.docx) - я могу помочь с парсингом
   - Markdown файла (.md) - структурированный формат

2. **Предоставить в виде таблицы Excel** с колонками:
   - ID | Title | Patient Info | Complaints | Anamnesis Morbi | Epidemiological History | Objective Data | Laboratory Data | Tasks | Reference Answer (можно в одном поле или разделить на 6)

3. **Просто скопировать текст задач сюда** - я помогу структурировать их в JSON формат

Рекомендую использовать JSON формат, так как он уже интегрирован в систему!
